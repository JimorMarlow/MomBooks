# Принудительное включение basic режима в HFS 3.x

## Проблема

На старых устройствах (например, iPad с устаревшей iOS) форма ввода пароля в современном интерфейсе HFS может не работать. Решение — использовать basic (упрощённый) HTML-интерфейс.

---

## Способ 1: Через веб-интерфейс

1. Откройте настройки HFS (шестерёнка или **Settings**)
2. Найдите раздел **Interface** или **UI**
3. Найдите опцию **"Default UI mode"** или **"Force basic UI"** → установите **basic**
4. Сохраните настройки (**Save**)

> ⚠️ В некоторых сборках HFS 3.x эти опции могут отсутствовать. В этом случае используйте Способ 3.

---

## Способ 2: Через файл конфигурации

Файл конфигурации HFS 3.x обычно находится в:

- `C:\Users\%USERNAME%\AppData\Roaming\HFS\hfs.json`
- Или в папке, где установлен HFS

Нужно добавить/изменить параметры:

```json
{
  "ui": {
    "defaultMode": "basic",
    "forceBasic": true
  }
}
```

После редактирования перезапустите HFS.

---

## Способ 3 (рекомендуемый): Кастомный index.html

Поскольку у вас есть скрипт `update_books_index.py`, который генерирует страницу с книгами:

1. В HFS 3.x зайдите в настройки виртуальной файловой системы (**Virtual File System**)
2. Для папки `books` отключите **"Auto-index"** или **"Directory listing"**
3. Положите сгенерированный `index.html` в `hfs/books/`

Тогда при переходе на `/books/` HFS будет отдавать ваш `index.html`, а не свой интерфейс.

### Преимущества этого способа:

- ✅ Работает на любых устройствах, включая старые iPad
- ✅ Не требует ввода пароля для просмотра (если чтение открыто для всех)
- ✅ Полный контроль над внешним видом страницы
- ✅ Загрузка файлов остаётся доступной через ссылки в вашем HTML

---

## Примечание

Если вы используете авторизацию, но хотите открыть чтение без пароля:

- В настройках HFS для папки `books` установите:
  - **Read**: `everyone` (или пустое значение — доступ без пароля)
  - **Upload**: по желанию, можно оставить только для авторизированных пользователей
