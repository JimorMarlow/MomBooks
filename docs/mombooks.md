# Файловый сервер для старого iPad

У мамы самый первый iPad, на который уже нельзя устанавливать программы. Safari устарел и не может открывать современные файлы для получения почты. Мне нужно закачивать ей книги в формате epub.

## Сервер

Для загрузки книг используется Python-скрипт `start_server.py` с базовой HTTP-аутентификацией (логин/пароль). Он позволяет в локальной сети подключиться к папке /hfs и показывает все содержимое в виде книжной полки. На файлы с расширением epub мама будет нажимать и загружать себе книги в iBooks. Я буду сам скачивать книги и раскладывать по папкам и авторам.

## Требования

Нужно создать скрипт на Python update_books_index.py, который будет сканировать все содержимое папке /hfs/books и создавать файл index.html как начальную страницу сервера. Нужно построить таблицу в виде книжной полки:
папка: книга1 книга2 ...

в самом верху полоса "Новое:" где показаны новые книги за последние 2 недели.
Ниже папки по алфавиту.
В самом низу полоса с книгами в корневой папке.

В индекс попадают только файлы с расширением epub.

Скрипт должен сделать обложку книги, снизу подпись: автор и название. Если удасться найти обложку в интернете, то положить в папку hfs/images и дать ссылку на нее. Если обложки нет, то сгенерировать изображение книги: сверху автор, по центру крупно название, снизу год. Данные взять из epub. Сгенерированные обложки класть в ту же папку hfs/images, имена обложек пусть совпадают с именами файла epub.

## Использование

После загрузки новых книг:
1. Запустить `python update_books_index.py` (или `start_books_server.bat` для автоматического запуска)
2. Запустить сервер: `python start_server.py`
3. Открыть в браузере: `http://<ваш-ip>:8080/` (логин: books, пароль: 3011)

Для новых книг, для которых нет обложки нужно загрузить или создать обложку и обновить индексный файл.
